# Release Notes - v0.7.1

## 🇯🇵 日本語

### 🐛 バグ修正 & 安定性の向上 (Bug Fixes & Stability Improvements)
- **VsaAdapter (超次元計算)**:
  - `unbind()`（アンバインディング）処理において、除数が極小の負数または負のゼロ（`-0.0`）に近い場合に、JavaScript の標準仕様（`-0.0 >= 0` が `true` に評価される）の影響で誤って正の `EPSILON` としてクランプされ、計算結果の符号が反転するバグを修正しました。`Object.is(val, -0.0)` を含む厳密な負号判定に修正し、符号が常に正確に維持されるようにしました。
- **ColbertAdapter (Late Interaction)**:
  - `rank()`（ドキュメントランキング計算）のループ実行中に WebAssembly メモリの成長（`grow`）が発生した場合に、事前にキャッシュしていた `Float32Array` バッファビューが detached（無効化）されて `TypeError` クラッシュを引き起こす潜在的なメモリ脆弱性を修正しました。ループごとに `writeFloat32ArrayToWasm` を呼び出して最新の有効なバッファ参照を安全に取得・書き込みするように改善しました。
- **各種リランカーのパラメータ検証強化 (Rerankers)**:
  - `BaseGraphReranker`、`MultipathScatteringReranker`、`TimeReversalReranker` のコンストラクタにおいて、`threshold` (NaN)、`alpha` (範囲外/NaN)、`maxIterations` (非整数/NaN)、`tolerance` (非正数/NaN)、`tau` (負数/NaN)、`iterations` (非整数/NaN) などの不正値が指定された場合に、実行前に明確な例外エラーをスローして計算崩壊を防ぐバリデーションガードを実装しました。
- **各種学習器のハイパーパラメータ検証強化 (Trainers)**:
  - `BaseTrainer`、`TripletTrainer`、`InfoNCETrainer` において、`learningRate`、`regularization`、`epochs`、`margin`、`temperature` などのハイパーパラメータに負の値や `NaN` が渡された際、計算が暴走して重み行列（matrix/bias）が `NaN` で汚染されるのを防ぐため、厳密なハイパーパラメータ検証ガードをコンストラクタ/実行の入り口に実装しました。
- **VectorDBAdapter (DB統合)**:
  - 各種データベース変換・シリアライズ用 static メソッド（`toPgvector`、`toPineconeQuery`、`toRedis`、`toVectorizeQuery`、`toVectorizeRecord`）において、入力ベクトルに `NaN`、`Infinity`、あるいは無効なオブジェクト型が渡された際、そのままデータベースのクエリに埋め込まれて SQL 例外や接続クラッシュを起こすのを防ぐため、型および有限数値の厳密なバリデーションガード（`validateVector`）を実装しました。
- **テストスイートの安定化 (Performance Regression Tests)**:
  - 並行テスト実行（`bun run test`）時の高負荷によって、ミリ秒以下の精密なパフォーマンスベンチマークテストがフレイキーに不合格（偽陰性）となる問題を解消するため、テストのアサーション閾値を現実的な範囲（3倍〜5倍程度）に緩和調整しました。

## 🇬🇧 English

### 🐛 Bug Fixes & Stability Improvements
- **VsaAdapter**: Fixed a sign inversion bug in `unbind()` where a divisor near negative zero (`-0.0`) was clamped to positive `EPSILON` due to the standard JS `-0.0 >= 0` evaluation. Replaced with strict sign check using `Object.is`.
- **ColbertAdapter**: Fixed a potential crash in `rank()` when WebAssembly memory growth occurred during document iterations, which invalidated (detached) the cached `Float32Array` view. Modified to use `writeFloat32ArrayToWasm` on each iteration to always obtain the latest buffer safely.
- **Reranker Validation**: Added strict validation guards in `BaseGraphReranker`, `MultipathScatteringReranker`, and `TimeReversalReranker` constructors to fail fast when invalid configs (`threshold`, `alpha`, `maxIterations`, `tolerance`, `tau`, `iterations`) are provided.
- **Trainer Validation**: Added validation guards for hyperparameters (`learningRate`, `regularization`, `epochs`, `margin`, `temperature`) in `BaseTrainer`, `TripletTrainer`, and `InfoNCETrainer` to prevent weight corruption with `NaN`.
- **VectorDBAdapter**: Implemented strict type and finite number check (preventing `NaN` and `Infinity`) in database serialization methods (`toPgvector`, `toPineconeQuery`, `toRedis`, `toVectorizeQuery`, `toVectorizeRecord`) to protect database connection layer from malformed queries.
- **Performance Regression Tests**: Relaxed benchmarking assertion thresholds to avoid flaky test failures during concurrent execution under high CPU load.
