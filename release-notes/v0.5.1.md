# ⚡ WarpVector v0.5.1 — SIMD Acceleration for Quantization / 量子化処理のSIMD高速化

WarpVector `v0.5.1` is a **performance update** that dramatically accelerates Vector Quantization through WebAssembly SIMD (128-bit parallel instructions).

WarpVector `v0.5.1` は、WebAssembly の SIMD (128ビット並列命令) を活用し、ベクトル量子化を劇的に高速化する**パフォーマンスアップデート**です。

---

## 🇬🇧 English

### 🚀 Performance Improvements

#### 1. Int8 Quantization (Nearly 3x Faster)
- The core algorithm for Int8 scalar quantization (`quantizeToInt8Wasm`) has been completely rewritten to utilize SIMD instructions (`f32x4.nearest`, `i32x4.trunc_sat_f32x4_s`, `i8x16.narrow_i16x8_s`, etc.).
- It eliminates scalar rounding and branching, processing 16 vector dimensions simultaneously without any branch mispredictions.
- Throughput increased from **~322K vecs/sec** to **~928K vecs/sec**.

#### 2. Binary Quantization (Bit-Packing) Accelerated
- The `quantizeToBinaryWasm` function was also optimized using SIMD instructions (`f32x4.gt`, `i32x4.bitmask`).
- Bit-packing now employs parallel evaluation and inline shift operations, evaluating 16 floating-point values into 2 packed bytes natively at the CPU register level.

### ✅ Testing & Validation

- **297 tests** across 44 files — All Passed
- No precision degradation. Absolute bit-exact output is guaranteed for both Static and Dynamic Int8 scaling, as well as Binary packing.

---

## 🇯🇵 日本語

### 🚀 パフォーマンスの改善

#### 1. Int8 量子化の劇的な高速化 (約3倍のスループット)
- Int8 スカラー量子化のコアアルゴリズム (`quantizeToInt8Wasm`) を SIMD 命令 (`f32x4.nearest`, `i32x4.trunc_sat_f32x4_s`, `i8x16.narrow_i16x8_s` 等) を用いて完全に書き換えました。
- 従来のスカラー丸め処理やループ内の条件分岐を排除し、16次元分を一度に並列処理することでキャッシュミスと分岐予測ペナルティをなくしました。
- これにより、スループットが **約 322K vecs/sec** から **約 928K vecs/sec** へと劇的に向上しています。

#### 2. Binary 量子化 (ビットパッキング) の最適化
- `quantizeToBinaryWasm` 関数にも SIMD 命令 (`f32x4.gt`, `i32x4.bitmask`) を導入しました。
- 符号の判定を並列化し、シフト演算を用いたインラインパッキング処理によって、16個のFloat値をわずかなCPUサイクルで2バイトの `Uint8` に圧縮して出力するようになりました。

### ✅ テストと精度検証

- **297 テスト** / 44 ファイル — 全パス
- 精度の劣化は一切ありません。静的/動的 Int8 スケーリングおよびバイナリパッキングの両方で、従来の処理と完全に同一（ビットレベルで一致）の出力結果が保証されています。
