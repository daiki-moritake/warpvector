# ⚡ WarpVector v0.6.1 — Workers, WebGPU, and Tracing / ワーカー、WebGPU、トレーシング

WarpVector `v0.6.1` is a **feature update** that introduces Isomorphic multithreading, GPU compute acceleration, and OpenTelemetry tracing.

WarpVector `v0.6.1` は、Isomorphicなマルチスレッド処理、GPUコンピュートアクセラレーション、および OpenTelemetry トレーシングを導入する**機能アップデート**です。

---

## 🇬🇧 English

### 🚀 Added

#### 1. New Packages
- **`@warpvector/worker`** — Web Worker and Node.js `worker_threads` support for Isomorphic multithreading.
  - Implements a generic `WarpWorkerPool` to parallelize batch processing.
  - Features `IsomorphicWorker` abstraction for seamless execution in both browsers and backend.
- **`@warpvector/gpu`** — WebGPU Compute Shader acceleration.
  - Introduces `WebGpuIntentAdapter` to process massively parallel vector transformations directly on the GPU.
- **`@warpvector/opentelemetry`** — OpenTelemetry tracing integration.
  - Exposes `OpenTelemetryTracer` which implements the `WarpTracer` interface for distributed tracing and observability.

### 🐛 Fixed

- **WebGPU Memory Leak**: Explicitly destroy temporary GPU buffers after GPU execution in `WebGpuIntentAdapter` to prevent VRAM exhaustion.
- **Worker Initialization Bug**: Fixed an issue where calling `WarpWorkerClient.init()` did not propagate the initialization state.
- **Worker Pool Edge Case**: Fixed an edge case in `WarpWorkerPool.terminate()` where active workers returning late responses could corrupt internal state.
- **GPU Thread Bounds**: Fixed WGSL `@workgroup_size` dimension mismatch in `WebGpuIntentAdapter` to ensure proper thread execution.
- **Worker Environment Check**: Relaxed browser environment check in `IsomorphicWorker` to properly support Service Workers and nested Web Workers.

## 🇯🇵 日本語

### 🚀 新機能

#### 1. 新規パッケージの追加
- **`@warpvector/worker`** — Isomorphic なマルチスレッド処理を実現する Web Worker および Node.js `worker_threads` サポート。
  - バッチ処理を並列化するための汎用的な `WarpWorkerPool` を実装。
  - ブラウザとバックエンドの両方でシームレスに実行できる `IsomorphicWorker` 抽象化を提供。
- **`@warpvector/gpu`** — WebGPU コンピュートシェーダーによるアクセラレーション。
  - 大規模並列でのベクトル変換を GPU 上で直接処理するための `WebGpuIntentAdapter` を導入。
- **`@warpvector/opentelemetry`** — OpenTelemetry トレース統合。
  - 分散トレーシングおよびオブザーバビリティのための `WarpTracer` インターフェースを実装する `OpenTelemetryTracer` を公開。

### 🐛 バグ修正

- **WebGPU メモリリーク**: `WebGpuIntentAdapter` における GPU 計算実行後、テンポラリの GPU バッファを明示的に破棄するように修正し、高負荷時の VRAM 枯渇やブラウザクラッシュを防止しました。
- **ワーカー初期化バグ**: `WarpWorkerClient.init()` の実装が空だったために初期状態がワーカーに伝搬されないバグを修正しました。
- **ワーカープール エッジケース**: `WarpWorkerPool.terminate()` の実行時に、遅れて届くワーカーからのレスポンスが内部状態を破損させるエッジケースを修正しました。
- **GPU スレッド境界**: `WebGpuIntentAdapter` において、GPU のスレッド境界を正しく確保するため WGSL の `@workgroup_size` の次元の不一致を修正しました。
- **ワーカー環境判定**: `IsomorphicWorker` におけるブラウザ環境の判定条件を緩和し、Service Worker やネストされた Web Worker を適切にサポートできるようにしました。
