# ⚡ WarpVector v0.5.0 — Async Pipeline & API Refinement / 非同期パイプライン・API 改善

WarpVector `v0.5.0` is a **major release with breaking changes** focused on making the pipeline API fully async-safe, unifying error handling, and improving type safety.

WarpVector `v0.5.0` は、パイプライン API の完全な非同期安全化、エラーハンドリングの統一、型安全性の向上に焦点を当てた**破壊的変更を含むメジャーリリース**です。

---

## 🇬🇧 English

### ⚠️ Breaking Changes

#### 1. Pipeline methods are now async

`run()`, `runBatch()`, `runAndFormat()`, and `dryRun()` now return Promises and require `await`.

```diff
- const result = pipeline.run(vector, { intent: "tech" });
+ const result = await pipeline.run(vector, { intent: "tech" });

- const results = pipeline.runBatch(vectors, { intent: "tech" });
+ const results = await pipeline.runBatch(vectors, { intent: "tech" });

- const formatted = pipeline.runAndFormat(vector, { format: "pgvector" });
+ const formatted = await pipeline.runAndFormat<string>(vector, { format: "pgvector" });

- const debug = pipeline.dryRun(vector, { intent: "tech" });
+ const debug = await pipeline.dryRun(vector, { intent: "tech" });
```

**Why**: `autoInit` internally uses async WASM initialization. Previously, `run()` was sync but `autoInit` was async, creating an inconsistency that could cause silent failures. Now all execution methods are async-safe.

#### 2. Error messages unified to English

All structured error messages are now in English. Error codes (`DIMENSION_MISMATCH`, `NOT_INITIALIZED`, etc.) and `instanceof` chains are unchanged.

| Before | After |
|--------|-------|
| `入力ベクトルの次元が一致しません` | `Dimension mismatch. Expected 1536, got 768.` |
| `初期化が完了していません` | `Not initialized. Call await pipeline.init() first.` |

#### 3. `runAndFormat<T>()` generic type parameter

```typescript
// Type-safe formatting
const pgStr = await pipeline.runAndFormat<string>(vec, { format: "pgvector" });
const pinecone = await pipeline.runAndFormat<{ vector: number[] }>(vec, { format: "pinecone" });
```

#### 4. Deprecated `QuantizationAdapter` static methods removed

```diff
- import { QuantizationAdapter } from '@warpvector/extras';
- const dist = QuantizationAdapter.hammingDistance(a, b);
- const dot = QuantizationAdapter.int8DotProduct(a, b);

+ import { hammingDistance, int8DotProduct } from '@warpvector/core';
+ const dist = hammingDistance(a, b);
+ const dot = int8DotProduct(a, b);
```

#### 5. `WarpPipeline.inputDim` is now read-only

`inputDim` was changed from a public property to a getter. Reading is unchanged; external writes will now cause a compile error.

#### 6. `flattenMatrix` error type unified

`flattenMatrix()` now throws `WarpDimensionMismatchError` instead of generic `Error`, enabling consistent `instanceof` error handling across the entire library.

### ✅ Testing

- **297 tests** across 44 files — All Passed
- **1,579 expect()** calls
- No regressions

## 🇯🇵 日本語

### ⚠️ 破壊的変更

#### 1. パイプラインメソッドの非同期化

`run()`, `runBatch()`, `runAndFormat()`, `dryRun()` が `Promise` を返すようになり、`await` が必要です。

```diff
- const result = pipeline.run(vector, { intent: "tech" });
+ const result = await pipeline.run(vector, { intent: "tech" });

- const results = pipeline.runBatch(vectors, { intent: "tech" });
+ const results = await pipeline.runBatch(vectors, { intent: "tech" });
```

**理由**: `autoInit` が内部で非同期 WASM 初期化を使用しているため。以前は `run()` が同期、`autoInit` が非同期という不整合がありました。すべての実行メソッドを非同期にすることで解消しています。

#### 2. エラーメッセージの英語統一

構造化エラーのメッセージがすべて英語に統一されました。エラーコード（`DIMENSION_MISMATCH` 等）と `instanceof` チェーンは変更ありません。

#### 3. `runAndFormat<T>()` ジェネリクス導入

戻り値に型パラメータを指定可能になりました。デフォルトは `unknown` です。

#### 4. `QuantizationAdapter` deprecated メソッド削除

`hammingDistance()`, `int8DotProduct()` は `@warpvector/core` から直接インポートしてください。

#### 5. `inputDim` の読み取り専用化

外部からの書き込みはコンパイルエラーになります。読み取りは変更なしです。

#### 6. `flattenMatrix` エラー型統一

`WarpDimensionMismatchError` をスローするようになり、ライブラリ全体で一貫した `instanceof` ハンドリングが可能になりました。

### ✅ テスト

- **297 テスト** / 44 ファイル — 全パス
- **1,579 expect()** コール
- リグレッションなし

### マイグレーション手順

1. **依存関係を更新**: `npm install warpvector@0.5.0`
2. **`pipeline.run()` 等に `await` を追加** — TypeScript コンパイラが `Promise<OutputVector>` 型で警告するため検出は容易
3. **`QuantizationAdapter.hammingDistance()` / `.int8DotProduct()` を置換** — `@warpvector/core` からインポート
4. **エラーメッセージの文字列比較を使用している場合は更新** — `instanceof` チェーンは影響なし
