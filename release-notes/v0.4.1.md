# 🔧 WarpVector v0.4.1 — Infrastructure & Quality Improvements / インフラ・品質改善

WarpVector `v0.4.1` is a maintenance release focused on infrastructure improvements, npm metadata completeness, and documentation updates. No source code changes — only configuration, CI/CD, and documentation fixes.

WarpVector `v0.4.1` は、インフラ改善、npmメタデータの完備、ドキュメント更新に焦点を当てたメンテナンスリリースです。ソースコードの変更はなく、設定・CI/CD・ドキュメントの修正のみです。

---

## 🇬🇧 English

### ✨ Highlights

- **Zero source code changes** — all improvements are infrastructure, config, and documentation
- **npm discoverability** — all 10 sub-packages now include `repository`, `homepage`, and `bugs` fields
- **CI now runs on `develop` branch** — previously only triggered on `main`
- **Migration Guide updated** to cover v0.3→v0.4 breaking changes

### 🔧 Infrastructure & DX

| Change | Details |
|--------|---------|
| Lock file unification | Standardized on `bun.lock`, removed `package-lock.json` |
| CI branch triggers | Added `develop` to push/PR triggers |
| CI type-check | Unified to `bun run typecheck` (was `npx tsc`) |
| npm metadata | `repository`, `homepage`, `bugs` added to all sub-packages |
| Playground | Integrated into npm workspaces, deploy updated to use `bun` |
| Engine constraints | `"node": ">=20.0.0"` added to root `package.json` |
| Release notes | 11 files reorganized into `release-notes/` directory |
| Articles | `docs/articles/` merged into `articles/launch/` |

### 🐛 Bug Fixes

- Removed phantom `@warpvector/ml` dependency from `@warpvector/rerank` (declared but unused)
- Fixed `@warpvector/extras` description (removed stale "ColBERT" reference)
- Added missing `license` and `author` fields to `create-warpvector-app`

### 📚 Documentation

- **Migration Guide** (EN/JA): Added v0.2→v0.3 and v0.3→v0.4 sections
- **assembly/README.md**: New file documenting WASM build flow

### ✅ Testing

- **297 tests** across 44 files — All Passed
- **1,579 expect()** calls
- No regressions
