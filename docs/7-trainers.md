# Online Learning Engines (Trainers)

In standard vector search engines, the vectors generated by pre-trained models (OpenAI, Cohere, etc.) are treated as the "correct answers" and used as-is for searching. However, in actual domain-specific searches (for example, your company's FAQ search or searching for jargon in a specific industry), that vector space is not always optimal.

WarpVector includes built-in "Trainers (learning engines)" that can **re-train and optimize the vector space in real-time within Edge/Node environments, directly from user feedback (click logs or likes/dislikes)**, without needing to spin up separate Python servers or GPU instances.

It comes equipped with an internal **Adam Optimizer**, enabling it to independently perform gradient descent equivalent to deep learning frameworks.

## 1. TripletTrainer (Triplet Learning)

This is the simplest form of Contrastive Learning.
It learns using a set of three vectors: an "Anchor (the search query)", a "Positive (the correct document clicked by the user)", and a "Negative (an irrelevant document that was displayed but not clicked)".

It automatically updates the space-warping matrix ($W$) and bias ($b$) so that the query gets closer to the positive and further away from the negative.

### Usage

```typescript
import { TripletTrainer } from 'warpvector';

const trainer = new TripletTrainer(1536);

// Current weights (initial state or previous state)
let currentWeights = {
  matrix: new Float32Array(1536 * 1536), // Initialize with an identity matrix, etc.
  bias: new Float32Array(1536)           // Initialize with a zero vector
};

// Call this when the user searches and clicks on one of the results
const updatedWeights = await trainer.updateOnline(
  currentWeights,
  {
    anchor: anchorVector,     // The search query vector
    positive: positiveVector, // The clicked correct document's vector
    negative: negativeVector,  // The ignored document's vector
  },
  { learningRate: 0.001 }
);

// Pass the updatedWeights to the IntentAdapter and use it for searching!
```

## 2. InfoNCETrainer (Contrastive Learning with Multiple Negatives)

This trainer uses a powerful loss function (InfoNCE Loss) that takes TripletLoss a step further, **simultaneously pushing away multiple incorrect answers (In-batch Negatives) against a single correct answer**.
It runs the standard method used in training modern search models (like SimCSE and CLIP) right on the edge.

It allows you to optimize the space faster and more accurately.

### Usage

```typescript
import { InfoNCETrainer } from 'warpvector';

const trainer = new InfoNCETrainer(1536);

// ... Obtain search and click events ...

// Train by passing one correct answer and a list of multiple incorrect answers
const updatedWeights = await trainer.updateOnline(
  currentWeights,
  {
    anchor: anchorVector,
    positive: positiveVector,
    negatives: [negativeVector1, negativeVector2, negativeVector3],
  },
  { learningRate: 0.001, temperature: 0.1 }
);
```

## 3. Trainer Use Cases

- **Search Result Personalization**: Learn the weights of an `IntentAdapter` uniquely tailored to each user based on their individual click history.
- **Domain Adaptation**: If the search accuracy for industry-specific jargon is low, operators can create a few pairs specifying "the correct answer for this query is this document" and have WarpVector learn them to correct the space.
- **Active Learning**: Feed the logs of like/dislike buttons directly into the learning pipeline, allowing the system to self-evolve.
