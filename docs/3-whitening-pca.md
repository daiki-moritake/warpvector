# Online Whitening & PCA

Vectors generated by large pre-trained language models (e.g., OpenAI `ada-002` or Cohere) have an inherent "spatial bias (anisotropy)". It is a phenomenon where all vectors point in a specific direction (a narrow cone-like region), causing the cosine similarity between seemingly unrelated sentences to be `0.80` or higher.

WarpVector's `WhiteningAdapter` uses **Oja's Rule** (an online Principal Component Analysis algorithm) to autonomously learn this "bias direction (first principal component)" from a stream of vector data, and dynamically removes that bias during search (Whitening/Isotropization).

## 1. How WhiteningAdapter Works

1. **Learning (Update)**: Every time a vector is sent to the system, it estimates the current "direction of maximum variance (bias)" through streaming learning. Because there is no need to hold all data in memory and perform Singular Value Decomposition (SVD), the memory footprint is extremely minimal.
2. **Application (Tune)**: During search or saving, the learned "bias direction" component is subtracted (orthogonally projected) from the vector.

This makes all vectors spherically (isotropically) distributed across the entire space, dramatically improving the resolution of cosine similarity (e.g., related items will clearly be `0.9` and unrelated ones `0.1`).

## 2. Basic Usage

```typescript
import { WhiteningAdapter } from 'warpvector';

// Setting to learn and remove the top 1 principal component (bias) for 1536-dimensional vectors
// learningRate is the update rate for online learning
const adapter = new WhiteningAdapter(1536, { 
  learningRate: 0.01, 
  numComponents: 1 
});

// Call 'update' alongside generating/saving vectors in your application
function onDocumentAdded(vector: number[]) {
  // Gradually learn the bias direction
  adapter.update(vector); 
}

// Call 'tune' during data search (or when saving to DB)
function onSearch(queryVector: number[]) {
  // Remove the bias component (canceling out meaninglessly high similarities)
  const whitenedQuery = adapter.tune(queryVector);
  
  // Search the Vector DB using whitenedQuery...
}
```

## 3. Why Online PCA over Batch PCA?

Normal PCA (Principal Component Analysis) requires loading millions of vector records into memory all at once and calculating a massive covariance matrix. This is unrealistic in Node.js or edge environments (like Cloudflare Workers).

WarpVector's Online PCA updates the principal component vector slightly every time one vector arrives, meaning:
- Memory usage is fixed to the size of a single vector (a few KB).
- Even if the data trend changes over time (concept drift), it automatically tracks and compensates for it.
- You literally just need to insert one line, `adapter.update()`, into the pipeline that adds data to your VectorDB.

## 4. Removing Multiple Components

By increasing `numComponents`, you can remove not just the first principal component, but also the second and third noise components. However, removing too many may result in the loss of the original meaning (signal) of the sentence, so `1` or `2` is usually recommended.

```typescript
const adapter = new WhiteningAdapter(1536, { numComponents: 2 });
```
